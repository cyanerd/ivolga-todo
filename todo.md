## 1. Медленная загрузка каталога

**Статус:** Готово

**Приоритет:** Критический

**Страница:** /catalog/

**Заказчик:** Критически важно. Сейчас даже тестировать сложно. Предложено: кеширование, CDN, прокачать сервер.

Первичная загрузка каталога очень медленная. Фотографии прогружаются все сразу и наслаиваются друг на друга. Нужно разобраться с причинами помимо веса фотографий: проверить lazy loading, количество загружаемых элементов, ресайз изображений на сервере, пагинацию/подгрузку.

**Решение:**
- В шаблоне каталога (`catalog.section/catalog-page/template.php`) у слайдов 2+ заменён `src` на `data-src` — браузер не загружает скрытые фото при первой отрисовке
- В CSS (`app.css`) добавлено правило `img[data-src] { display: none }` — скрытые слайды не рендерятся до загрузки
- В JS (`productCard.js`) переписан слайдер: при наведении на карточку подгружаются скрытые фото, через 5 сек после загрузки страницы все оставшиеся фото подгружаются фоном. После загрузки `data-src` удаляется → CSS-правило перестаёт действовать → картинка появляется
- Также на сервер загружены оптимизированные изображения (iblock)
- Итого: первая загрузка — 20 картинок вместо 60, трафик при отрисовке уменьшен в ~3 раза

## 11. Как назначить коллекцию вне артикула

**Статус:** Готово (24.02, анализ)

**Приоритет:** Низкий (документация)

**Заказчик:** Понятно, что группы товаров приходят из 1С. Вопрос: как создать вкладку (Новый год, Распродажа), НЕ связанную с группой товаров из 1С? Нужна ли для этого 1С? Может ли один товар быть в нескольких вкладках?

**Ответ заказчику:**

Коллекции на сайте — это свойство «KOLLEKTSIYA» у товаров. Оно приходит из 1С, но можно менять и в самом Битрикс.

**Как создать новую коллекцию (например, «Новый год» или «Распродажа»):**
1. Зайти в карточку любого товара в админке Битрикс
2. Найти свойство «Коллекция» (KOLLEKTSIYA)
3. Добавить новое значение — оно появится в списке для всех товаров
4. Проставить эту коллекцию нужным товарам

**Один товар может быть только в одной коллекции** — это ограничение текущей структуры (свойство хранит одно значение). Если нужно, чтобы товар был в нескольких коллекциях одновременно — потребуется доработка (сделать свойство множественным).

**Важно:** при следующей синхронизации 1С может перезаписать значение коллекции. Если нужны коллекции, которые 1С не трогает — можно создать отдельное свойство (например, «Акции»), не связанное с 1С.

## 13. Плавающий баг: цвета меняются местами

**Статус:** Готово (24.02)

**Приоритет:** Средний

**Страница:** /product/*

**Заказчик:** Не поправлено, баг сохраняется.

При первом открытии карточки товара и выборе цвета — квадраты с цветами меняются местами. При повторном выборе — перестают. Воспроизводится только при первом посещении карточки.

**Решение:** Причина — нестабильная сортировка в запросе `getVariantColors()` (`init.php`). Если у товаров одинаковые `DATE_ACTIVE_FROM` и `SORT`, MySQL возвращает их в произвольном порядке. При перезагрузке страницы порядок мог меняться, а после попадания в кеш — фиксировался. Добавлен `'ID' => 'ASC'` как детерминированный тайбрейкер в запрос + `ksort()` для алфавитной сортировки цветов.

**Заказчик:** Видит только заказы текущей сессии (все от 10.02.2026), предыдущие заказы отсутствуют.

**Анализ:** Компонент `sale.personal.order.list` показывает заказы по `USER->GetID()`. Проблема скорее всего в SMS-авторизации (`auth.php`): при каждом входе ищется пользователь по `PERSONAL_PHONE` через `NormalizePhone()`. Если формат телефона в базе не совпадает с нормализованным — пользователь не находится, создаётся **новый аккаунт**. Старые заказы остаются на старом ID.

**Возможные причины:**
1. `NormalizePhone()` возвращает разный формат в зависимости от входных данных (с «+7», «8», пробелами)
2. Телефон в базе мог быть записан до внедрения SMS-авторизации в другом формате
3. При каждой «неудачной» авторизации (п.19) мог создаться дубль пользователя

**Решение:** Нужно проверить в админке — есть ли дубли пользователей с одним телефоном. Если да — объединить заказы на один ID и добавить более надёжный поиск (например, по последним 10 цифрам телефона).
